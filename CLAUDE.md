# Content Curator — Claude Code Reference

## Project Overview

Personal content curation system with feedback loop and taste learning.
Self-hosted on Linux with Docker. No paid infrastructure.

## Workflow Maintenance Rule

**After every new feature or improvement to a workflow, update the JSON file in `workflows/` AND re-upload it to n8n via the API so the running instance is always in sync with the files in the repo.**

Upload command:
```bash
source .env && python3 -c "
import json
with open('workflows/vX_name.json') as f:
    w = json.load(f)
for field in ['id','createdAt','updatedAt','versionId','active','meta']:
    w.pop(field, None)
print(json.dumps(w))
" | curl -s "http://localhost:5678/api/v1/workflows/WORKFLOW_ID" \
  -H "X-N8N-API-KEY: \${N8N_API_KEY}" \
  -X PUT -H "Content-Type: application/json" -d @-
```

## Data Storage

- **Episode log** (`podcasts.episodes`): PostgreSQL, `postgres_shared` container, DB `homelab`, schema `podcasts`
- **Spotify show cache** (`podcasts.spotify_shows`): same schema
- **User profile / preferences**: `data/user_profile.json` (stays as JSON — preferences, not episode log)
- **Ratings history**: `data/ratings_history.json` (stays as JSON — feedback loop for Claude scoring)
- **n8n ↔ postgres**: both on `proxy-net` Docker network; n8n connects to `postgres_shared:5432`
- **norm() function**: `podcasts.norm(t TEXT)` — lowercase + unaccent + strip special chars. Used for all dedup checks.

To apply schema changes to running postgres:
```bash
source ../shared/.env
docker exec -i postgres_shared psql -U $POSTGRES_USER -d homelab < ../infra/postgres/init/00_schemas.sql
```

To migrate `podcast_queue.json` → DB (one-time, already done):
```bash
bash scripts/migrate_queue_to_db.sh
```

## Stack

- **n8n** (Docker) — workflow orchestration
- **Claude API** (`claude-sonnet-4-5`) — filtering, scoring, conversation, calendar suggestions
- **Telegram Bot API** — delivery, inline feedback buttons, free-text conversation
- **Spotify API** (free OAuth2) — onboarding: followed podcasts + listen history
- **YouTube Data API v3** (free, 10k units/day) — video discovery
- **Listen Notes API** (free tier, 10k calls/month) — podcast discovery
- **Podcast Index API** (free, open) — podcast discovery + RSS lookup
- **iTunes Search API** (Apple, free, no key) — podcast discovery
- **Apple Calendar via CalDAV** (iCloud, read-only) — consumption scheduling
- **RSS** — known podcast feeds + website feeds
- **SMTP** — optional email delivery

## Spotify OAuth2 PKCE Flow

1. Generate `code_verifier` (43-128 random chars, URL-safe)
2. Compute `code_challenge = BASE64URL(SHA256(code_verifier))`
3. Build auth URL: `https://accounts.spotify.com/authorize?response_type=code&client_id=...&scope=user-follow-read+user-read-recently-played+user-library-read&redirect_uri=...&code_challenge_method=S256&code_challenge=...`
4. User opens URL, authorizes, gets redirected to `redirect_uri?code=AUTH_CODE`
5. Exchange: POST `https://accounts.spotify.com/api/token` with `grant_type=authorization_code`, `code`, `redirect_uri`, `client_id`, `code_verifier`
6. Response: `access_token` + `refresh_token` → store in `data/spotify_token.json` (chmod 600)
7. Refresh: POST same endpoint with `grant_type=refresh_token`, `refresh_token`, `client_id`

## CalDAV iCloud App-Specific Password

iCloud CalDAV requires an **app-specific password**, NOT your Apple ID password.
Generate at: https://appleid.apple.com → Security → App-Specific Passwords → +

Use `https://caldav.icloud.com` as base URL. PROPFIND with `Depth: 1` to list calendars.
REPORT with `calendar-query` and `time-range` to fetch events.

Only `PROPFIND` and `REPORT` methods are allowed. Never `PUT`, `DELETE`, `POST`.

## Security Model

- n8n bound to `127.0.0.1:5678` — not exposed to internet directly
- All external traffic via reverse proxy (ngrok/cloudflare tunnel) with HTTPS
- Telegram webhook validates `X-Telegram-Bot-Api-Secret-Token` on every request
- n8n protected by basic auth
- All secrets in `.env` only, never hardcoded, never logged
- `data/` is gitignored — no tokens or ratings leave the machine
- n8n runs as user `1000:1000`, no privileged mode
- Spotify tokens stored in `data/spotify_token.json` with `chmod 600`
- CalDAV credentials never leave the server, HTTPS only
- Telegram: validate `chat_id === TELEGRAM_CHAT_ID`, reject unknown senders silently

## Rate Limiting

- All external APIs: retry with exponential backoff (3 attempts, delays: 1s, 2s, 4s)
- YouTube: guard against quota (10k units/day) — log usage, abort if near limit
- Spotify: respect 429 responses with `Retry-After` header
- Podcast Index: HMAC-SHA1 auth required on every request

## Profile Update Rules

```
👍 quick like:    item_score += 0.5, topic_weight += 0.2
👎 quick dislike: item_score -= 0.5, topic_weight -= 0.1
⭐ full rating:   item_score = (topic_score + host_score) / 2
proactive positive: topic_weight += 0.5, host_affinity += 0.3 (if host mentioned)
proactive negative: topic_weight -= 0.2
survey completion: podcast priority = f(frequency + completion_rate)
All weights: clamped [1, 10]
Recalculate discovery search terms after every 5 profile updates
```
